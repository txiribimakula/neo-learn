How do controls decide how much width to take?

You drop two boxes and want each one at 50%. Or a fixed label next to a field that fills whatever is left. Behind that intuition there are just four ways to ask for width. See them all here, live — and find out they're plain flexbox.

Max — takes the whole row by itself (100%)
Fixed — a number of pixels, no matter what
Percent — a fraction of the container
Remaining — fills whatever space is left
↓ Play on the left while you read
01 · the starting point

Every control is born taking the max

When you insert a new control it asks for 100% of the width by default: it sits alone on its row and pushes the next one below. It's the safest behavior — nothing gets squeezed — and from there you decide whether to share the row.

Hit ＋ Add control a couple of times and watch each one stack under the previous one.

/* Max = a whole row just for me */
.field { flex: 0 0 100%; }
02 · pixels that don't move

The fixed width: you set the pixels

Sometimes you want an exact width: a 90px button, a 120px label. The control is always the same size even if the container grows or shrinks. Select a control, pick Fixed and drag the slider.

To see it: drag the “container” slider up top. Fixed doesn't flinch; everything else does.

.button { flex: 0 0 90px; }
03 · a fraction of the whole

The percentage: scales with the container

Here's your “each one at 50%”. A control at 50% is always half the container: if it goes from 700 to 900px, the control grows with it. Two at 50% fill the row; three don't fit and the third drops to a new line.

Drag the “container” slider and notice: percentages keep their proportion while everything else changes size.

.col { flex: 0 0 50%; }
04 · whatever is left is mine

The remaining: fills the leftover

This is the clever one. It doesn't ask for a concrete width: it grabs everything left over in its row. A fixed 120px label + a remaining field = the field stretches by itself to the edge, whatever the container measures.

The rule: a remaining control owns its row — it can share it with at most one fixed control and nothing else. Anything else (a percentage, a second fixed, another remaining) jumps to a new line. That way a remaining always has a clear, predictable leftover to fill.

/* basis 0 + grow 1 = "give me the rest" */
.field { flex: 1 1 0; }
05 · how a row fills up

The packing rules, step by step

When several controls share a row, the container fills it like this:

1 · Place fixed and percent controls left to right while they fit — those already know their size.
2 · A max control always takes a whole row by itself.
3 · A remaining control grabs the leftover of its row — it shares that row with at most one fixed control and nothing else; everything else starts a new row.
4 · If something doesn't fit, it wraps to the next line.

06 · the secret

It's flexbox, nothing more

No magic: each row is a display:flex row, and every mode is a recipe for the flex shorthand (grow · shrink · basis):

.row { display:flex; }

/* Max */ flex: 0 0 100%;
/* Fixed */ flex: 0 0 120px;
/* Percent */ flex: 0 0 50%;
/* Remaining */ flex: 1 1 0;
