Git worktrees

Un worktree te permite tener varias carpetas de trabajo del mismo repositorio, cada una en una rama o commit distinto, compartiendo la misma historia Git.

1base de objetos Git compartida
Ncarpetas de trabajo independientes
0stashes obligatorios para cambiar de contexto

Checkout normal

Con una sola carpeta, cambiar de rama reemplaza el contenido de esa carpeta. Si tienes cambios sin commit, necesitas resolverlos antes de moverte.

git checkout feature/login

Una carpeta

Solo hay un checkout activo. La rama cambia, pero la carpeta es la misma.

Cambios locales

Los cambios sin commit pueden bloquear el checkout si chocan con la rama destino.

Coste mental

Tienes que recordar que contexto estabas usando antes de cambiar.

Sin worktrees

  1. 1Estas en `main` dentro de una unica carpeta.
  2. 2Quieres revisar un hotfix o una feature.
  3. 3Stash, commit temporal o checkout bloqueado si hay cambios incompatibles.
  4. 4Vuelves a `main` y recuperas contexto manualmente.

Con worktrees

  1. 1`main` sigue abierto en su carpeta.
  2. 2Creas otra carpeta para `feature/login` o `hotfix/crash`.
  3. 3Cada carpeta tiene su propio index y sus propios cambios.
  4. 4La historia Git se comparte, asi que no duplicas el repositorio completo.
# Una sola carpeta: cambiar de rama cambia ese workspace
git checkout feature/login
# Si hay cambios locales incompatibles, Git te frena.
